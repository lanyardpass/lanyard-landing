# Guide image pipeline (landing site)

How images get prepared for `/guides`. Two kinds: **stills** (most images) and
**layered-depth banners** (the parallax effect). This is the *web* pipeline; the
app's collectible/perk image rules live in the workspace `docs/IMAGE_PIPELINE.md`.

---

## Golden rules (both kinds)

1. **Process at the highest resolution you have. Downscale exactly once, last.**
   Every downscale is lossy, so never run masking / inpainting / compositing on
   an already-shrunk copy. Do the work big, shrink at the end with Lanczos.
2. **Compress once.** Commit a **lossless** (or near-lossless) master and let
   `astro:assets` do the single delivery compression to webp/avif. Don't ship a
   lossy master that Astro then re-compresses (double loss).
3. **Commit masters at the largest size you'd ever serve, not the raw original.**
   Full-bleed banner master ≈ 2200px; in-column ≈ 1600px. The ~350 MB Inbox
   originals never get committed; only the derived masters do.
4. **`astro:assets` (`<Image>`/`<Picture>`) generates the responsive widths**
   from the master; the Netlify edge image service serves per-device sizes.

---

## Stills

Use `astro:assets`. Put the master under `src/assets/guides/<guide>/`, import it,
render via `<Image>` (or the `Figure` component). To prep a master from a big
original, a single Lanczos downscale is all you need:

```js
sharp(original).rotate()                       // honor EXIF orientation
  .resize({ width: 2200, fit: 'inside', withoutEnlargement: true })
  .jpeg({ quality: 82, mozjpeg: true })        // or .webp({ quality: 90 })
  .toFile(out)
```

`.rotate()` with no args auto-orients from EXIF (this is how `IMG_4418` got
straightened). Never upscale.

## Layered-depth banners (`LayeredParallax`)

The Isle-of-Berk effect: one photo split into three planes that drift at
different rates on scroll, giving real depth. Generated by
`tools/make-parallax-layers.py`:

```
pip3 install "rembg[cpu]" pymatting scipy pillow numpy   # one-time
python3 tools/make-parallax-layers.py \
  "Inbox/Epic Guide Photos/IMG_4407.jpeg" \
  src/assets/guides/epic/berk
```

It outputs three lossless webp masters:

| Plane | What it is | Notes |
|---|---|---|
| `front.webp` | the salient subject (statues) | crisp cutout, alpha matting |
| `village.webp` | everything else, sky removed, subject painted out behind it | the sharp scene |
| `sky.webp` | a sharp sky plane filling behind everything | **never blurred** |

Wire them into the guide:

```mdx
<LayeredParallax back={sky} mid={village} front={front} alt="…" caption="…" wide />
```

### Hard-won gotchas (these are why the first two attempts looked wrong)

- **Resolution does NOT fix the cutout edge.** `rembg`/u2net resize to 320px
  internally, so the mask shape is model-limited. Crisp edges come from
  `isnet-general-use` + **alpha matting**, not from more pixels.
- **Keep the sky plane SHARP.** It's the sky the reader actually sees. An early
  version blurred it (left over from when it was a hidden fill) and the sky
  looked fake.
- **The sky mask must protect tall foreground.** A naive blue/brightness key eats
  the hazy top of a mountain/spire, punching holes that reveal the plane behind.
  The fix: a *tight* "clearly blue" threshold, kept to blobs **connected to the
  top edge**, so it can't bleed down into the subject.
- **Paint the subject OUT of the background.** If any background plane still
  contains the subject, the moving sharp subject ghosts against it. A
  distance-transform "nearest real pixel" fill removes it cleanly; the fill sits
  behind the front cutout, so it's only ever glimpsed as clean texture on motion.
- **Motion is downward-only, and zero at rest.** Drifting down means a receding
  plane only ever opens a gap at the *top* (where the sky belongs), never a hole
  at the bottom. Starting at translate 0 means the first thing you see is the
  full, uncropped image.
- **The subject barely separates from the mid plane.** The strong depth is the
  village receding against the near-static sky; the subject gets a small extra
  pop. Pushing the subject too far exposes the painted-out area behind it.

Tuning lives in `src/components/guides/LayeredParallax.astro`: `range` (overall
travel) and each layer's `data-depth`. All motion honors
`prefers-reduced-motion` (planes sit still = the clean composite).
